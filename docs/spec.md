# Command Center 設計書(公開版)

## コンセプト

AI社員の稼働状況を可視化する静的ダッシュボード。目的は3つ:
①「誰が・何を・どこで止まり・何が完成したか」を10秒で把握 ②タスク・ログ・成果物を構造化データとして資産化 ③運用の証拠をそのままポートフォリオにする。

設計原則: KISS / YAGNI / データ駆動(HTML直書き禁止) / 見せかけ禁止(実データのみ)。

## MVP範囲

- 社員6名(司令塔・クリップ係・リサーチ係・編集係・ナレッジ係・開発係)
- 1ページ構成、JSON5本+build-log.md、手動更新(AI起案→人間確認→push)
- 対象外(将来): GitHub Actions自動化・Issue連携・KPI・週報生成

## データ契約

スキーマは `AGENTS.md` を正とする(agents / tasks / runs / outputs / knowledge-index)。
runs.jsonは追記専用。blockedタスクはblocked_reason必須。

## ステータス

| status | 意味 | 表示 |
|---|---|---|
| idle | 未着手・待命 | 灰バッジ |
| running | 稼働中 | 緑バッジ+点滅ドット |
| waiting | 人間確認・外部要因の待機 | 黄バッジ |
| blocked | 停滞(理由必須) | 赤バッジ+最上部の停滞パネルへ昇格 |
| completed | 完了(タスクのみ) | 青バッジ |
| archived | 保管 | 一覧に表示しない |

## 画面設計(1ページ・上から)

1. ヘッダー: タイトル/日付/サマリー数字4つ(稼働中・停滞・本日完了・総成果物)
2. 停滞パネル: blockedが1件以上のときだけ表示(タスク名・理由・next_action)
3. AI社員カード: 部署ごとにグループ化。カード内容=名前/部署/ステータスバッジ/現在タスク名/最終稼働/最新出力リンク。PC3列→タブレット2列→スマホ1列
4. 今日のタスク & 今日の成果物(2カラム、スマホでは縦積み)
5. Build Log: logs/build-log.md の最新20行をテキスト表示
6. ナレッジ更新状況: knowledge-index.jsonの件数と直近5件
7. フッター: next_action一覧/リポジトリへのリンク

## 実装要件

- 素のHTML/CSS/JS。外部ライブラリ・CDN・ビルド工程なし
- `fetch()` は相対パス(GitHub Pagesのサブパス配下で動作すること)
- JSON取得失敗時はセクションを空にして続行(画面全体を壊さない)
- 375px〜1280pxで横スクロールなし
- `prefers-color-scheme` によるダーク/ライト両対応
- 更新日時は `agents.json` の `updated` をヘッダーに表示

## 受け入れ基準(Done Criteria)

1. `data/agents.json` に7人目を追加すると、コード変更なしでカードが増える
2. タスクを `blocked` にすると停滞パネルが出現し、理由が表示される
3. スマホ幅(375px)で横スクロールが発生しない
4. ローカル静的サーバとGitHub Pagesの両方で動作する
5. JSONを1本削除しても他のセクションは表示され続ける

作業サイクル・公開安全ルール・社員追加手順は `AGENTS.md` を参照。
